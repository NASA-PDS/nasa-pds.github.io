LDD Change Control Board¶

Applicable only to Discipline Local Data Dictionaries, a PDS LDD Change Control Board will be formed to manage and monitor changes that occcur to dictionaries that could have impacts on various Discipline Node communities.

Membership¶

The LDD Change Control Board (CCB) will have a rotating chair from Discipline Node staff, with voting members consisting of up to one member from each discipline node. Additional discipline node members can participate in CCB discussions as lurkers, whereas they have no voting rights.

Current membership will maintained by the LDD CCB Chair and tracked by the PDS LDD CCB Github Team.

Communication¶

A Google Group or mailing list will be created for the LDD CCB, which will be managed and maintained by the LDD CCB Chair. All initial discussions and approvals will occur via the LDD CCB Google Group.

Voting and Approvals of Change Requests¶

Voting and approvals of proposed changes to an LDD will occur using the Github CODEOWNERS Github Pull Request “required reviews” feature.

After a Pull Request is created, the PDS LDD CCB will automatically be assigned as a reviewer for the pull requesta and a notification will be sent to the user.
